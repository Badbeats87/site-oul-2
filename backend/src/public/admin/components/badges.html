Usage Guidelines

Status Badges
  • Use for domain-specific status (LIVE, SOLD, RESERVED)
  • Place prominently in lists and cards
  • Use consistent colors aligned with business meaning
  • Make text uppercase for status badges
Category/Tag Badges
  • Use for genres, categories, labels
  • Keep text short (1-3 words)
  • Can be capitalized or mixed case
  • Group related badges together
Badge Sizing
  • Small: Inline with text, secondary information
  • Default: Standard use in lists and cards
  • Large: Hero sections, prominent callouts
Placement
  • Status badges: Top-right of card or row
  • Category badges: Below title or in metadata
  • Multiple badges: Wrap with consistent spacing
  • In tables: Right-aligned for consistency
Best Practices
  • Use color meaningfully (success = green, danger = red, etc.)
  • Not more than 3-4 badges per item (shows too much)
  • Ensure sufficient contrast with background
  • Use rounded pills (border-radius: full) for badges
Accessibility Checklist:
  • ✓ Sufficient color contrast (4.5:1 minimum)
  • ✓ Not conveyed by color alone (use dot, text, icon)
  • ✓ No hidden status information (visible text)
  • ✓ Semantic meaning in visual design
  • ✓ Consider adding aria-label if icon-only
  • ✓ Text readable at all sizes
  • ✓ Color-blind safe palette used
