Usage Guidelines

When to Use Each Type
  • Success: Action completed (save, submit, upload)
  • Danger: Errors or destructive actions (failed operations, deletions)
  • Warning: Caution situations (low inventory, unusual conditions)
  • Info: Helpful information (tips, notices, system status)
Alert Content
  • Keep messages clear and actionable
  • Use title for quick context, message for details
  • Include relevant information (item names, counts, etc.)
  • Don't use only color to convey meaning (use icons/text)
Dismissible Alerts
  • Success and info alerts can be dismissible
  • Error alerts should persist until resolved
  • Include close button (×) for dismissible alerts
  • Provide keyboard shortcut (Escape key)
Positioning
  • Place at top of form for validation errors
  • Place near affected element for contextual alerts
  • Success/confirm messages in prominent position
  • Stack multiple related alerts vertically
Duration
  • Errors: Persist until fixed (don't auto-dismiss)
  • Success: Auto-dismiss after 5-8 seconds (optional)
  • Warnings: Persist (important information)
  • Info/Tips: Dismissible (nice-to-know, not critical)
Accessibility Checklist:
  • ✓ Sufficient color contrast (4.5:1 minimum)
  • ✓ Not conveyed by color alone (use icons/text)
  • ✓ role="alert" for dynamic/auto-appearing alerts
  • ✓ Close button has aria-label
  • ✓ Focus managed when alert appears
  • ✓ Screen reader announces alerts immediately
  • ✓ Keyboard dismissible (close button accessible)
