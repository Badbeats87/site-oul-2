Button Component

Interactive element for user actions

Primary Buttons

Use for main actions. High emphasis, prominent call-to-action.

Default State
<button class="button button--primary">Primary Button</button>
Hover State
Hover shows darker background and shadow elevation
Focus State
Focus shows outline and box-shadow for accessibility
Disabled State
<button class="button button--primary" disabled>Disabled</button>
Accessibility: Primary buttons have sufficient contrast (7:1). Focus indicators are visible and distinct. Disabled state uses opacity to indicate non-interactivity.

Secondary Buttons

Use for alternative actions. Medium emphasis, less prominent than primary.

Default State
<button class="button button--secondary">Secondary Button</button>
With Other Buttons
Combine primary and secondary for action groups

Usage Guidelines

When to Use Primary Buttons
  • Main action on the page (Save, Submit, Send)
  • Call-to-action that moves user forward
  • Only one primary button per page section
When to Use Secondary Buttons
  • Alternative actions (Cancel, Close, Back)
  • Less important than primary action
  • Multiple secondary buttons acceptable
When to Use Size Variants
  • Small: Inline actions, compact layouts, table actions
  • Default: Standard UI, forms, dialogs
  • Large: Hero sections, important CTAs, mobile
  • Block: Full-width on mobile, forms
When to Use Color Variants
  • Success: Positive actions (approve, confirm, save)
  • Danger: Destructive actions (delete, remove)
  • Accent: Highlight, special attention
Accessibility Checklist:
  • ✓ Minimum 44x44px touch target
  • ✓ Visible focus indicator (outline or shadow)
  • ✓ Color not used alone to convey meaning (use text/icons)
  • ✓ Sufficient contrast (4.5:1 for text)
  • ✓ Clear, descriptive button text
  • ✓ Disabled state visually distinct
  • ✓ Keyboard accessible (Tab, Enter)
